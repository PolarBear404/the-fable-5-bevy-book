# UI 基础：Node 与布局

第 16 章排《渡口夜话》的时候，我们把文字分成了两类：**台上的**跟着镜头走，**玻璃上的**钉死在屏幕老位置。当时玻璃上只挂了一行字，还许了个愿——`Node` 的整套布局本事，第 28 章是正题。这一章兑现。

戏班的行话里，挂在戏园前厅、写着今晚戏码与角儿名号的那块牌子叫**水牌**。它不在台上，不归灯光布景管，观众一进门先看它。游戏里的血条、比分、菜单、提示语，全是这类东西：它们属于**界面**（UI），不属于场景——镜头推拉摇移，它们一动不动。本章请出戏班的水牌师傅，从零挂起一面完整的前厅看板。

承担这摊活的是 `bevy_ui` 与 `bevy_ui_render` 两个 crate（都在默认 feature 集合里，常用类型都在 `bevy::prelude`）。它的核心是一个组件加一套系统：

- **`Node`——一块归布局系统管的矩形**。你不给它 `Transform`，也不给它 `Sprite`；你填的是意图——“宽 300 像素”“占父级的一半”“居中”“留 16 像素内边距”——每帧由布局系统把这些意图结算成屏幕上的位置和尺寸。结算规则就是网页开发者用了十几年的 **Flexbox** 与 **CSS Grid**，Bevy 借道 [taffy](https://github.com/DioxusLabs/taffy) 库把这两套布局算法搬进了引擎；
- **一棵 UI 树**。`Node` 靠第 9 章的 `ChildOf`/`Children` 组成父子层级：父节点划地盘，子节点在地盘里排座次。整棵树最终画在一层透明玻璃上，盖在它所属相机的画面之上——“所属相机”四个字眼下先不深究，28.12 有专门一节。

没接触过网页布局也不要紧——本章假定你一句 CSS 都没写过，Flexbox 和 Grid 都从零讲起；写过的读者则会发现字段名几乎逐个对得上，直接把肌肉记忆搬过来就行。

按活儿排期，本章这么走：

- **第一块牌子**——最小的 UI 场景：一个 `Node` 一块色；顺带见识 UI 世界的第一个哑巴坑；
- **量尺的什么时候来**——布局系统在一帧里的位置，`ComputedNode` 账本的读法，物理像素与逻辑像素；
- **单位阅兵**——`Val` 的七种量法：像素、百分比、视口单位，以及 `UiScale` 这把全局旋钮只拨得动谁；
- **三层皮**——盒模型：margin、border、padding 各占一层，`BoxSizing` 决定图纸上的数字量到哪层，再开一盏透视镜亲眼看；
- **九宫对齐板**——Flexbox 的主轴与交叉轴，`justify_content` 与 `align_items` 两个旋钮逐档拨过去；
- **分地**——`flex_grow`、`flex_shrink`、`flex_basis`：一条街三户人家，余粮怎么分、亏空怎么摊；
- **在场与离场**——绝对定位钉四角，以及 `Visibility::Hidden` 与 `Display::None` 的关键区别：人走了，席位留不留；
- **谁贴在上面**——玻璃是一层层叠的：默认的叠放顺序、`ZIndex` 与 `GlobalZIndex` 各管哪一段；
- **首演座位表**——CSS Grid：先画格线后入座，跨行跨列，还有一次格线从 0 数起的当场翻车；
- **一张皮四种绷法**——`ImageNode` 把图片请上玻璃：拉伸、九宫格、平铺，第 15 章的 `TextureSlicer` 在这儿白送；
- **玻璃上的字**——UI 文本与布局的合作：字多板子自己长高，不许换行就捅出去，捅出去还能裁；
- **玻璃跟着哪台相机走**——多相机时 UI 归谁：默认规则、一枚官印、点名到户；
- **收场《前厅》**——给《打瓦》公演装一面响应式 HUD：拖窗自适应的比分牌、命图标、战利品架与中场横幅。

# Gizmos、诊断与开发工具

戏曲舞台上有个奇特的行当叫**检场**：一身便服，大摇大摆走上正在演出的台面，搬椅子、递道具、给武生垫毯子。台下看客有一条不成文的约定——**只当看不见他**。他明明在台上，却不属于戏。

这一章要给你的游戏请一位检场人。第 26 章调的画质是给玩家看的；这一章画的东西**只给你自己看**：把碰撞盒描出来、给速度画上箭头、在屏幕角落挂一块帧率水牌、往终端里记一本流水账。它们与游戏同台，却不是游戏的一部分——发布的时候，观众照旧看不见他。

上一章结尾我们对第 20 章的《打瓦》许了个愿：给它加一层“开发者之眼”。这一章兑现。在合龙之前，先把三样工具逐个练熟，它们分住三个 crate：

- **`bevy_gizmos`——检场的粉笔**。一套*即时模式*的画线 API：在任何系统里喊一声 `gizmos.line_2d(...)`，这一帧屏幕上就有一条线，下一帧不喊就没有。线框、圆圈、箭头、网格、还有一手描线字，全是一行一个；另有*保留模式*伺候大批量的静态线条，以及一只现成的、可以拽着实体满场跑的 Transform Gizmo 搬运把手；
- **`bevy_diagnostic`——场记的账本**。一套轻量的数值记录设施：帧率、帧耗时、实体口数、CPU 内存占用，每样都是一本带历史的账，可以念到终端，也可以自己立新账目；
- **`bevy_dev_tools`——开发工具箱**。FPS 水牌、可拖拽的诊断小窗、编辑器味道的无限网格地面、状态换场播报员……全是“挂上插件就能用”的成品。这个 crate **不在默认 feature 集合里**——检场人不上正式戏单，这扇门要自己开，本章会亲眼看一次没开门的报错。

按活儿排期，本章这么走：

- **第一道粉线**——`Gizmos` 系统参数与即时模式的心智模型：为什么粉线要每帧重画，忘了重画会怎样；
- **粉线字典**——2D/3D 绘制词汇全家福：格线、站位圈、走位弧、软垫框、落点叉，还有 builder 式的加参手法；
- **规格与分组**——`GizmoConfigStore` 拨线宽、线型、拐角；自定义 `GizmoConfigGroup` 给不同用途的粉线各配一路开关；顺带看一次忘了登记配置组的运行期翻脸；
- **会写字的粉线**——文本 gizmo：跟着实体跑的活字标签，以及它只认 ASCII 的哑巴脾气；
- **慢时钟下的粉线**——把固定时钟拨到每秒 4 拍，亲眼看 `FixedUpdate` 里画的粉线怎样在两拍之间*驻留*；
- **一劳永逸的粉线**——保留模式：几百根定位线只搭一次，存成资产挂上实体；
- **现成的描边**——一行组件描包围盒（`ShowAabbGizmo`）、一键给四种灯画形（`ShowLightGizmo`）；
- **场记的账本**——`Diagnostic` 的数据模型：一本账三种读数；三大内置账本插件与每秒念账的 `LogDiagnosticsPlugin`；
- **自记一笔**——`register_diagnostic` 立自己的账目，闭包懒求值，合账与重开；
- **开发工具箱**——`dev` feature 这扇门与没开门的编译报错；FPS 水牌、诊断小窗、状态播报员；
- **迷你检修间**——无限网格打底，点选道具、拽把手搬转缩：`TransformGizmoPlugin` 与第 25 章拾取的会师；
- **收场《检场》**——工作台版《打瓦》叠上一整层调试皮，一个插件管全部，摘一行就还原。

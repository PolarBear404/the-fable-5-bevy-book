> **状态：待执行。** 依据：`0.19-outline-assessment.md`（P-1～P-13 已批准落实）＋ `code/` 全库 API 命中实扫（2026-07-02）。

采用"先编译绿、再逐章深修"而非纯逐章：TextFont 等横切变化波及 5 章 25 个文件，按错误类型批量处理一次扫清，S2 各章专注行为、叙述与插图。S1 期间正文与代码短暂不同步——只存在于本分支，可接受。

## S2　逐章深度迁移（ch1 → ch21 顺序推进）

### 每章统一流程

1. **行为实测**：`cargo run` 该章 main.rs 与全部 examples，输出/画面逐一对照正文现有描述，记录差异
2. **代码定稿**：按该章工单处理章内专属变化；新增小节的配套代码（新 listing 或扩展现有）此时写入并编译验证
3. **正文改写**：受影响小节按"原生 0.19"改写（无痕原则）；输出示例块用实跑结果替换；新增小节成文；检查正文里引用的官方示例路径/API 名
4. **插图重建**（ch12 起）：跑 `scripts/make_chXX_figures.py`，逐图与正文描述核对；图文不符则修代码或修正文，不准 P 图
5. **构建与检查**：`mdbook build book` 绿；本章交叉引用、术语首现解释完整
6. **收尾**：PROGRESS.md 更新，commit `chXX: 迁移到 0.19`

**每章完成定义（DoD）**：check 绿 ＋ 全示例实跑与正文一致 ＋ 无痕 ＋ 插图重建核对 ＋ mdbook build 绿 ＋ 已提交。

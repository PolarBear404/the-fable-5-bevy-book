> 状态：已完成（2026-07-07 开工，2026-07-08 收尾）。本文件是本章全部决策与实测的现场记录。

## 范围与衔接

- OUTLINE 条目：bevy_gizmos(_render)（即时/保留模式调试绘制、文本 Gizmos、Transform Gizmo）、
  bevy_dev_tools（FPS overlay、无限网格、诊断浮层等）、bevy_diagnostic。示例：给 Breakout 加调试层。
- 衔接：ch26 收尾预告原文「画质是给玩家看的，调试可视化是给你自己看的——Gizmos 即时画线画框、
  FPS 浮层、诊断数据，给第 20 章的 Breakout 加一层"开发者之眼"」——本章逐项兑现。
  ch28 是 UI 基础（Node/布局）：本章 FPS overlay / DiagnosticsOverlay 内部是 UI，只用不讲，
  一句话指路 ch28（沿用 ch26 状态牌的处理）。TransformGizmo 复用 ch25 的 MeshPickingPlugin +
  `On<Pointer<Click>>`（回访）；FixedUpdate 清屏上下文回访 ch18；3D 场景用 ch21/22 概念。
- 大纲出入：无。按大纲执行。
- **Breakout 调试层的落地方式**：ch20-breakout 是二进制 crate、组件全私有，跨 crate 引用需给它
  加 lib 并把组件改 pub——侵入已完稿章节，不做。改为在 ch27 crate 里放一个「工作台版《打瓦》」
  （court+凳+球+瓦阵+FixedUpdate 玩法内核，剥去菜单/音效/记分，常量与配色照抄 ch20），调试层
  作为独立 plugin 叠上去。正文明说「同一个插件原样搬回你的完整版《打瓦》也能用」。
- **demo 判定：不配**。调试可视化的教学点靠静态开/关对比图即可传达；唯一真交互点
  （transform gizmo 拖拽）是编辑器旁支非本章核心，且 ch25 已有拖拽类 demo 先例覆盖手感；
  不为旁支扛 26MB。FixedUpdate 步进 vs 平滑的对比考虑用一张动图（运动即教学点，同 ch18 尺度）。

## 示例设计

**主题**：《检场》——戏班宇宙的收官检修夜。戏曲舞台上的**检场人**：穿便服上台搬道具、观众约定俗成
"看不见"他——正是调试层的完美隐喻：画在台上、不进正式演出（release 不带 dev feature）。
角色：检场（新角，画粉线/搬道具/挂水牌）、场记（账本=诊断，沿用）、老雷（班主，验收）。
道具延续：道具箱（2D 滑动目标）、后台仓库（3D 场景：箱、灯）、《打瓦》工作台版（收场）。

**crate**：`ch27-dev-tools`。features：`default=["dev-tools"]`，`dev-tools=["bevy/bevy_dev_tools"]`
（crate 转发 feature，同 ch24/25 手法；--no-default-features 复现 E0432 报错）。

**Listing 规划（编号随写作微调）**：

## 1　范围与衔接

**大纲条目（OUTLINE ch25）**：bevy_picking（mesh/sprite/UI 拾取、指针事件与 Observer 集成、
连击计数 `Pointer<Click>::count`）、bevy_camera_controller（现成相机控制器）；
示例：点选/拖拽 3D 物体 + 自由视角相机。**与大纲无出入。**

**0.19 API 底账（已逐项核对 vendor/bevy）**：

- **插件与 feature 格局**（root Cargo.toml:133-170、bevy_internal/default_plugins.rs:102）：
  - 默认 profile `2d`/`3d`/`ui` 都带 `picking` = ["bevy_picking","mesh_picking","sprite_picking",
    "ui_picking"]——**picking 默认全开**，无需 Cargo 动手
  - `DefaultPlugins` 注册 `DefaultPickingPlugins`（= PointerInputPlugin + PickingPlugin +
    InteractionPlugin，lib.rs:287），但这只是"管线"，**不含任何 backend**
  - backend 注册三家三样：`SpritePickingPlugin` 在 SpritePlugin 里自动注册（bevy_sprite/lib.rs:108）、
    `UiPickingPlugin` 在 UiPlugin 里自动注册（bevy_ui/lib.rs:179）、**`MeshPickingPlugin` 必须手动
    add**（raycast 有 CPU 成本，官方所有 picking 示例都手动加）
  - re-export：`bevy::picking`（bevy_internal/lib.rs:71）、`bevy::camera_controller`（:33）；
    prelude 里有 MeshPickingPlugin/Pointer/Click/Drag/Pickable 等（bevy_picking prelude）；
    FreeCamera 一族**不在 prelude**，须 `use bevy::camera_controller::free_camera::...`
  - `free_camera`/`pan_camera` feature **默认不开**（camera_controller/lib.rs:16 "all off by
    default"）→ 本章 crate 开 bevy features = ["free_camera","pan_camera"]
- **拾取管线四段**（bevy_picking/lib.rs:101-153 + PickingSystems 枚举 :258）：
  Input（First）→ ProcessInput/Backend/Hover/PostHover/Last（都在 PreUpdate）。
  指针（pointer.rs）→ backend 报 PointerHits（backend.rs）→ hover 合并排序出 HoverMap
  （hover.rs）→ pointer_events 派发高层事件（events.rs:667）。observer 全部在 PreUpdate 的
  同步点里 resolve——Update 里的系统看到的是"事件处理完之后"的世界
- **`Pointer<E>` 事件形状**（events.rs:71-87）：`#[entity_event(propagate = PointerTraversal,
  auto_propagate)]`，同时也是 Message（可 MessageReader 读）与 Component。字段：`entity`（目标）、
  `pointer_id`（PointerId：Mouse/Touch(u64)/Custom(Uuid)）、`pointer_location`（Location：
  target+position 窗口坐标）、`event: E`（内层数据，`Deref` 直达）。
  `PointerTraversal`（events.rs:94-125）：先父链（ChildOf），根实体后跳到**窗口实体**再停
- **事件家族 17 门**（events.rs，InteractionPlugin add_message 清单 lib.rs:428-444）：
  - 悬停：`Over`/`Out`（无限冒泡）、`Enter`/`Leave`（**只发给"非共享祖先"**，带
    is_in_bounds/was_in_bounds，对标 web mouseenter/mouseleave；Pointer::propagate=false 造出
    不冒泡效果，events.rs:85 注释）、`Move`（带 delta，屏幕像素）
  - 按压：`Press`/`Release`（带 button: PointerButton::Primary/Secondary/Middle）、`Click`
    （带 duration + **count 连击计数**；Press 也带 count）
  - 拖拽：`DragStart`→`Drag`（delta+distance）→`DragEnd`（distance）；另一侧 `DragEnter`
    （dragged）/`DragOver`/`DragDrop`（dropped）/`DragLeave`——**拖拽者与被拖过者两套视角**
  - 滚轮：`Scroll`（unit/x/y/hit/phase）；取消：`Cancel`（触摸/系统级）
  - 帧内顺序（pointer_events 文档 :619-625）：Out→Leave→DragLeave；DragEnter→Enter→Over；
    每移动 DragStart→Drag→DragOver→Move；每按键 Press 或 Click→Release→DragDrop→DragEnd→DragLeave
  - **Click/Release 打给上一帧悬停实体**（:656-660，触摸松手当帧无悬停）
- **连击**：`PickingSettings.multi_click_interval`（默认 Duration::from_millis(500)，lib.rs:354）；
  PointerButtonState.clicking 记 (Instant, u8)（events.rs:479）。PickingSettings 其余开关：
  is_enabled/is_input_enabled/is_hover_enabled/**is_window_picking_enabled**（默认全 true）
- **窗口拾取**（window.rs）：指针在窗口上→窗口实体永远垫底命中（order=NEG_INFINITY、depth=0、
  无 normal）→"点空处"事件路由给 Window 实体——给窗口挂 observer 可做"点空白取消选中"
- **`Pickable` 组件**（lib.rs:196-254）：`should_block_lower`（挡不挡下层）+ `is_hoverable`
  （自己收不收）；默认 true/true；`Pickable::IGNORE` = false/false；**没这组件的实体默认能收
  也能挡**（对 mesh backend；sprite backend 例外，见下）
- **HitData**（backend.rs:135）：camera/depth/position（Option，世界空间——窗口 backend 给的是
  屏幕坐标 extend(0.0)）/normal（Option，未必归一化——缩放 mesh）/extra
- **mesh backend**（mesh_picking/mod.rs）：`MeshPickingPlugin` + `MeshPickingSettings`
  { require_markers: false, ray_cast_visibility: VisibleInView }；require_markers=true 时要
  `MeshPickingCamera`（相机）+ `Pickable`（实体）双标记；RenderLayers 过滤（:105）
- **MeshRayCast**（mesh_picking/ray_cast/mod.rs:210）：SystemParam；
  `cast_ray(ray: Ray3d, settings: &MeshRayCastSettings) -> &[(Entity, RayMeshHit)]` 近到远排序；
  MeshRayCastSettings { visibility, filter: &dyn Fn(Entity)->bool, early_exit_test }；
  RayMeshHit（point/normal/barycentric_coords/distance/triangle/uv/triangle_index）；
  Ray3d 来自 `Camera::viewport_to_world`（ch13 旧识）。AABB 剔除→mikktspace 无关、走
  Möller-Trumbore（ray_mesh_intersection）。**性能提示在正文点一句：每 mesh 全三角形遍历，
  大场景该上物理引擎的加速结构**（官方 mesh_picking.rs 头注同款口径）
- **hover 状态派**（hover.rs）：`PickingInteraction`（Pressed=2/Hovered=1/None=0，**backend 报上来
  就有份，含没有 Pickable 组件的实体**——update_interactions 只写"有该组件的实体"，:226 组件要
  自己挂）、`Hovered(pub bool)`（含子孙，immutable 组件、要自己挂才更新）、`IsDirectlyHovered`
  （不含子孙）；`PointerInteraction`（pointer.rs:73，挂在**指针实体**上的全命中清单，
  get_nearest_hit() 拿最近——官方示例画命中点+法线 gizmo 的数据源）
- **sprite backend**（bevy_sprite/picking_backend.rs）：**`Regardless of this setting, only
  sprites marked with Pickable will be considered`（:56）——sprite 要手动挂 Pickable 才可拾取，
  与 mesh 默认全可拾正好相反**（教学对比点）；`SpritePickingSettings { require_markers: false,
  picking_mode: AlphaThreshold(0.1) }`；`SpritePickingMode::BoundingBox` / `AlphaThreshold(f32)`
- **UI backend**（bevy_ui/picking_backend.rs）：自动注册；Node 天生参与（focus 系统同源）；
  本章只以 Text 牌子最小引入（官方 simple_picking.rs 手法），Node/布局正题在 ch28
- **FreeCamera**（camera_controller/free_camera.rs）：`FreeCameraPlugin`（系统挂
  RunFixedMainLoop 的 BeforeFixedMainLoop，ch18 讲过这个位置）+ `FreeCamera` 组件
  `#[require(FreeCameraState)]`（ch03 required components 旧识）。字段（Default :139-165）：
  sensitivity 0.2 / key_forward..down WASD+EQ / key_run ShiftLeft / mouse_key_cursor_grab
  **Right（按住抓）** / keyboard_key_toggle_cursor_grab KeyM（切换抓）/ key_snap_reverse
  ControlLeft + axis_top/right/front Numpad7/3/1（轴对齐吸附，SampleAutoCurve 平滑转过去）/
  walk_speed 5.0 / run_speed 15.0 / scroll_factor 0.04879016（≈ln 1.05，**指数**调速：每格滚轮
  ×e^factor）/ friction 40.0（松键后 smooth_nudge 指数刹车）/ rotation_speed PI/16*60 /
  vertical_movement_axis World|Local（Bevy/UE/Blender vs Unity/Godot 之别）。
  state.initialized 首帧从 Transform 读 yaw/pitch 并 info! 打键位表；抓取时
  CursorGrabMode::Locked + 光标隐藏；鼠标输入走 **AccumulatedMouseMotion**（ch17 讲过 raw 摇臂
  ——**网页 iframe 里恒为 0，除非 pointer lock**，demo 判定关键）；pitch clamp ±PI/2；
  **single_mut()：场里只能有一台 FreeCamera 相机**（:279）
- **PanCamera**（pan_camera.rs）：`PanCameraPlugin` + `PanCamera` 组件（**无 require，自足**）。
  字段：enabled / zoom_factor 1.0 (min 0.1, max 5.0) / zoom_speed 0.1 / key_zoom_in/out =/− /
  pan_speed 500.0 / key_up..right WSAD / rotation_speed PI + key_rotate_ccw/cw QE /
  mouse_pan_settings { enabled: true, button: **Left** }——鼠标拖平移**用的就是
  Pointer<Drag> 事件 + 窗口 observer**（pan_camera.rs:15 引 bevy_picking::events，
  add_window_observer :34）——官方自家先示范了"picking 事件当相机输入"，收场示例师出有名；
  zoom 改的是 Transform.scale（正交相机习惯，ch13 旧识）
- 官方示例第一参考：picking/{simple,mesh,sprite,dragdrop,debug}_picking.rs、
  camera/{free,pan}_camera_controller.rs

**微实验点数（每概念至少一拨）**：

2. On<Click> 忘包 Pointer→编译错（25-2）
3. Hovered 组件不挂不更新 vs PickingInteraction 同样要挂（25-3，双组件对照）
4. multi_click_interval 500ms→120ms：三连点变三个单击（25-4 拨表）
5. propagate(false) 注释掉→父货架重复收单（25-5）
6. 窗口 observer 收"点空处"（25-5）
7. Pickable 四象限纱幕（25-6 数字键 1-4）
8. drag delta y 方向反直觉（25-7，往上拖 delta.y 为负——屏幕系）
9. RayCastVisibility::VisibleInView→Any：隐身漆盒点得中（25-9 拨档）
10. SpritePickingMode AlphaThreshold(0.1)→BoundingBox：点立牌脚边透明角（25-10 拨档）
11. UI 牌子 should_block_lower false→点穿牌子选中身后展品（25-11）
12. FreeCamera walk_speed/friction/scroll_factor 手感三连拨（25-12，官方示例键位）
13. require_markers=true→没标记全场失灵，补 MeshPickingCamera+Pickable 复活（25-6 或 25-9 带）

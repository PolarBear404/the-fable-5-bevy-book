## 1　范围与衔接

**大纲条目（OUTLINE ch23）**：bevy_gltf：GltfAssetLabel、WorldAssetRoot、GltfMaterial 与
#Material0/std 材质标签、按命名取实体、与 DCC 工具（Blender）的工作流；示例：加载带动画的角色模型。

**0.19 API 底账（已逐项核对 vendor/bevy）**：
- glTF 场景加载为 `WorldAsset`（bevy_world_serialization），挂场景组件是
  `WorldAssetRoot(Handle<WorldAsset>)`（components.rs:23，require Transform+Visibility）；
  实例就绪后根实体得 `WorldInstance`，并对根实体触发 EntityEvent `WorldInstanceReady { entity, instance_id }`
  （world_asset_spawner.rs:33）
- `GltfAssetLabel`（label.rs）：Scene/Node/Mesh/Primitive{mesh,primitive}/Texture/
  Material{index,is_scale_inverted}/DefaultMaterial/Animation/Skin/InverseBindMatrices；
  `from_asset(path)` 拼标签路径
- `Gltf` 总资产（assets.rs）：scenes/named_scenes、meshes/named_meshes、materials/named_materials、
  nodes/named_nodes、skins/named_skins、animations/named_animations、default_scene、source
  （include_source 时才有）；GltfMesh/GltfNode/GltfPrimitive/GltfSkin 均为独立 Asset
- **材质两本账**：bevy_gltf 只出 `GltfMaterial`（引擎无关，标签 `Material{N}`）；bevy_pbr 以
  GltfExtensionHandler 身份（bevy_pbr/src/gltf.rs）为每个材质追加 `Material{N}/std` 标签的
  `StandardMaterial` 子资产（转换函数 `standard_material_from_gltf_material`），网格实体挂的
  MeshMaterial3d 指向 /std 这份；DefaultMaterial 同理有 `DefaultMaterial/std`
- 命名落到实体：节点实体得 `Name`（无名则 GltfNode{i}）；网格图元实体得 `Name`（primitive_name）
  与 `GltfMaterialName`/`GltfMeshName`；场景根得 `Name` + `GltfSceneName`；灯实体得 `Name`。
  extras 落 `GltfExtras`/`GltfSceneExtras`/`GltfMeshExtras`/`GltfMaterialExtras`
- `GltfLoaderSettings`（loader/mod.rs:185）：load_meshes/load_materials（RenderAssetUsages）、
  load_cameras=true、load_lights=true、load_animations=true、include_source=false、
  default_sampler/override_sampler、validate、convert_coordinates、skinned_mesh_bounds_policy；
  用 `asset_server.load_builder().with_settings(|s: &mut GltfLoaderSettings| ...).load(...)`（ch14 已教 load_builder）
- `GltfConvertCoordinates`（convert_coordinates.rs）：glTF 前方 +Z / Bevy 前方 −Z；
  `rotate_scene_entity`（转场景根）与 `rotate_meshes`（转网格资产）两开关，默认都关；
  实验性（文档有 CAUTION）；相机与灯例外（本来就按 Bevy 约定）。全局默认在 `GltfPlugin::convert_coordinates`
- 动画三件套（bevy_animation 已核）：`AnimationGraph::from_clip(Handle<AnimationClip>) -> (graph, AnimationNodeIndex)`
  （graph.rs:445）、`AnimationPlayer::play(index) -> &mut ActiveAnimation`（lib.rs:866）+ `.repeat()`、
  `AnimationGraphHandle(Handle<AnimationGraph>)` 组件（graph.rs:138）。**推进/写姿势两个系统的 Query
  都必需 &AnimationGraphHandle——漏挂 = 时间与姿势一起冻住，零报错零警告**（前世 0.18 已实证机制，
  本章须在 0.19 重新实测）
- 灯与相机的生成细节（loader/mod.rs 1569-1831）：gltf 相机节点 spawn `Camera3d + Projection +
  Camera { is_active: 文件内第一台为 true }`——玩家自己的相机若同存即 order 冲突；点光按
  KHR 规范 candela，Bevy 换算 `intensity = candela × 4π`（流明），range 缺省 20，灯实体是
  节点实体的**子实体**、Name 取 light 名（非节点名）
- 官方示例第一参考：examples/gltf/load_gltf.rs（WorldAssetRoot 最小加载）、
  examples/animation/animated_mesh.rs（WorldInstanceReady observer + 放动画）、
  examples/gltf/edit_material_on_gltf.rs（GltfMaterialName 换材质）、load_gltf_extras.rs、
  query_gltf_primitives.rs、update_gltf_scene.rs
